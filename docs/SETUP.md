# ProjectEcho - New PC Setup Guide

Follow these steps when cloning this project on a new computer.

## Prerequisites

- **PHP 8.2+** with these extensions enabled:
  - `extension=gd` (for QR code generation)
  - `extension=curl`
  - `extension=fileinfo`
  - `extension=intl`
- **Composer** installed globally
- **Node.js & npm** installed
- **XAMPP** or similar (for local development with Apache/MySQL)

## Step 1: Enable PHP GD Extension

The project requires PHP's GD extension for image/QR code handling.

### On XAMPP (Windows):

1. Open `C:\xampp\php\php.ini` in a text editor
2. Find the line `;extension=gd` (should be around line 931)
3. Uncomment it by removing the semicolon: `extension=gd`
4. Save the file
5. Restart Apache from XAMPP Control Panel

### Verify:
```bash
php -i | grep -i gd  # Should show GD support enabled
```

## Step 2: Install PHP Dependencies

```bash
composer install
```

This installs Laravel framework, Filament admin panel, QR code libraries, and all other PHP packages.

**Time:** ~2-5 minutes depending on internet speed.

## Step 3: Environment Setup

```bash
copy .env.example .env
php artisan key:generate
```

This creates your `.env` configuration file and generates a unique encryption key for the application.

## Step 4: Database Setup

```bash
php artisan migrate
```

This creates the SQLite database file (`database/database.sqlite`) and runs all migrations to set up tables.

## Step 5: Install Frontend Dependencies

```bash
npm install
```

This installs Tailwind CSS, Vite, and other frontend build tools.

**Time:** ~1-2 minutes.

## Step 6: Build Frontend Assets

```bash
npm run build
```

This compiles your CSS and JavaScript using Vite. Creates optimized assets in `public/build/`.

For development with hot-reload:
```bash
npm run dev
```

## Step 7: Start the Development Server

```bash
php artisan serve
```

The app will be available at `http://127.0.0.1:8000`

## Important Notes

### QR Code Signing Keys

- QR codes are cryptographically signed using **Ed25519** keys stored in `storage/app/flightplan-qr-keys/`
- **New keys are auto-generated** on first run
- **Old QR codes from a different PC will be invalid** — they were signed with different keys
  - To use old QR codes: Copy the `.pem` files from the old PC
  - Otherwise: Generate fresh QR codes on this PC

### Subsequent Development Sessions

After initial setup, to resume development:

```bash
php artisan serve  # Start web server
npm run dev        # Start frontend watcher (optional, for live reload)
```

### Troubleshooting

**"PHP extension ext-gd is missing"**
- See Step 1: Enable PHP GD Extension

**"No application encryption key has been specified"**
- Run: `php artisan key:generate`

**"Database file does not exist"**
- Run: `php artisan migrate`

**"Vite not recognized" during `npm run build`**
- Run: `npm install` first

**QR codes aren't scanning/invalid signature**
- This is expected if you're using QR codes from a different PC
- Generate new flight plans on this PC to create valid QR codes

### Development Tips

- **Database reset:** `php artisan migrate:fresh` (deletes all data and recreates tables)
- **Clear caches:** `php artisan optimize:clear`
- **Tinker (REPL):** `php artisan tinker` to test code interactively
- **File upload dir:** `storage/app/public/` (symlinked to `public/storage/`)
