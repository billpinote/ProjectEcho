# QR Error Reference

These errors are shown when a scanned Echo flight plan QR code cannot be accepted by the current application.

## Error #01

Meaning: The scanned text does not have the expected Echo QR segment count.

Expected V2 offline payload shape:

```text
ECHOFPL|2|OFFLINE|K1|S1|flight_id|issued_at|payload|signature
```

Common causes:

- The QR image was not decoded completely.
- The scanned QR is not an Echo flight plan QR.
- The payload was copied or pasted with missing `|` separators.

## Error #02

Meaning: The scanned text has the right number of segments, but one or more required envelope fields are invalid.

The app checks:

- Prefix is `ECHOFPL`
- Version is `2`
- Mode is `OFFLINE`
- Key ID is `K1`
- Schema ID is `S1`
- Flight ID is numeric
- Issued timestamp uses `YYYYMMDDTHHMMSSZ`

Common causes:

- The QR came from an unsupported Echo version, mode, key, or schema.
- The QR data was altered.
- The scanned text is not an Echo V2 offline payload.

## Error #03

Meaning: The signature segment is not valid base64url data.

Common causes:

- The QR was damaged or incompletely scanned.
- The signature text was manually edited.
- Extra characters were introduced while copying the payload.

## Error #04

Meaning: The payload looks like an Echo V2 offline QR, but the Ed25519 signature cannot be verified with the current public key.

Common causes:

- The QR was generated by an older sandbox key.
- The QR was generated in a different environment with a different key pair.
- The payload was modified after it was signed.
- The current environment is missing the matching verification key.

Security note: Do not bypass this validation. If old QRs must remain valid, import the trusted old public key or add an explicit key-rotation compatibility path.

## Error #05

Meaning: The signature is valid, but the embedded flight plan snapshot cannot be decoded.

Common causes:

- The compressed payload data is corrupted.
- The payload data does not match schema `S1`.
- The QR was generated by code using incompatible snapshot encoding.

## Default Unsupported Payload Message

If the app cannot classify the failure into one of the numbered QR errors, it falls back to:

```text
Expected a valid Echo QR payload. V2 signed offline payloads and legacy V1 database payloads are supported.
```

This usually means the scanned text is not a supported Echo QR payload.

## Valid QR. Needs ATC Review.

This is not an error.

Meaning: The QR signature is valid and the embedded flight plan data was decoded, but the app could not find a matching live flight plan row in the current database.

Common causes:

- The QR was generated from another environment or branch.
- The QR is valid, but the flight plan has not yet been imported into this database.
- The QR contains an offline signed copy that still needs ATC review.

User action:

- ATC/ATMO should review the displayed flight plan details before accepting, rejecting, or otherwise processing the flight plan.
